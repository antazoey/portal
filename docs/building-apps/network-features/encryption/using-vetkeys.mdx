# vetKeys API demo

<MarkdownChipRow labels={["Advanced", "Encryption"]} />

This demo provides a canister (`src/system_api`) that offers the [proposed vetKD system API](https://github.com/dfinity/interface-spec/pull/158), implemented for demonstration, testing, and initial development purposes. We stress that it does not offer security yet, as no threshold cryptography is used yet to secure the key.

This demo uses files found in [the examples repository](https://github.com/dfinity/examples/tree/master/rust/vetkd).

### Step 2: Start a local instance of the replica.

First, [start the local replica](/docs/building-apps/developer-tools/dfx/dfx-start) if necessary and [create](/docs/building-apps/developer-tools/dfx/dfx-new) or open a project.

### Step 3: Ensure that dfx uses the canister IDs that are hard-coded in the source code:

```sh
dfx canister create system_api --specified-id s55qq-oqaaa-aaaaa-aaakq-cai
```

Without this, `dfx` may use different canister IDs for the `system_api` and `app_backend` canisters.

### Step 4: Ensure that the required node modules are available in your project directory:

```sh
npm install
```

### Step 5: Register, build, and deploy the project:

```sh
dfx deploy
```

This command should finish successfully with output similar to the following one:
